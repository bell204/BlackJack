개발동기 
객체지향을 이해하는데는 뷰와 데이터베이스가 연결된 게시판보다, 도메인 규칙이 충분한 카드 게임이 좋을 것 같아서 구현해보았습니다.

또한 TDD에 대해 이해하기에도 좋을 것 같아서,
테스트 주도 개발(채수원 저)를 참고하여
TDD를 하며 게임을 구현해보았습니다.
 

<h2>테스트 순서</h2>

1 실패하는 테스트 코드 (예시)

2 Test 결과 화면(빨강)

3 테스트에 통과할만한 작은 코드 작성(예시)

4 리팩토링 코드 (예시)

3 객체 추상화

4 TDD 하면서 구현  

5 리팩토링
